# 개별종목 보유 ETF 분석기

국내 상장 ETF 전체의 구성종목(PDF)을 모아, **개별 상장주식이 ETF들에 의해
얼마나 보유되고 있는지**를 집계해 엑셀로 만듭니다. (병렬 크롤링 버전)

## ★ 먼저: KRX 로그인 설정 (필수, 한 번만)
2025-12-27 KRX 개편으로 데이터 조회에 로그인이 필요합니다.
1. https://data.krx.co.kr 에서 **무료 회원가입**
2. 같은 폴더의 **`krx_계정.txt`** 를 메모장으로 열어 1줄=아이디, 2줄=비밀번호 저장

## 실행
`실행.bat` 더블클릭 (또는 `python etf_holdings.py`).
실행하면 **기준일과 비교 기준일을 물어봅니다** — 원하는 날짜(YYYYMMDD)를 치거나,
그냥 엔터치면 기본값(기준일=최근 영업일, 비교일=2025-12-30)으로 진행합니다.
끝나면 같은 폴더에 `ETF편입분석_기준일_vs_비교일.xlsx` 생성.
(매번 묻는 게 싫으면 상단 `ASK_DATES = False` 로 두면 CONFIG의 날짜를 그대로 씁니다.)

## 속도
- ETF별 상세 기능 추가로 캐시 버전이 올라가, 다음 1회는 두 날짜 모두 재크롤링됩니다(이후 캐시).
- 이제 ETF PDF를 **동시에 여러 개(기본 8개)** 받아서 처음 실행도 대략 20~40분대.
- 한 번 받은 날짜는 `cache/` 에 저장 → 다음 실행부터 그 날짜는 안 받습니다.
  (특히 비교 기준일 2025-12-30 은 한 번만 받으면 끝)
- KRX가 자주 차단/오류를 내면 `etf_holdings.py` 상단 `WORKERS` 를 4~6으로 낮추세요.

## 산출물 (엑셀 시트 7개)
- **1_시총대비편입비중상위** : 시총 대비 ETF 편입 비중이 큰 종목 상위 300
- **2_편입비중급증상위** : 비교 기준일 대비 편입 비중이 늘어난 종목 상위 300
- **3_ETF보유액상위** : ETF 보유액(원)이 큰 종목 상위 300
- **4_ETF보유액급증상위** : 비교 기준일 대비 ETF 보유액이 늘어난 종목 상위 300
- **5_조회** : ★ A열 노란 칸에 **종목코드(예 005930)나 종목명**을 한 줄에 하나씩 여러 개 입력 →
  각 행 가로로 종목코드·종목명·현재/기준일 편입비중·변화·ETF보유액·시총·보유ETF수가 자동으로 뜸
- **7_종목별ETF조회** : ★ 노란 칸에 종목코드/종목명 입력 → 그 종목을 보유한 **ETF 전체가 아래로 나열**.
  각 ETF의 AUM · 현재 비중/보유액 · 기준일 비중/보유액 · 증감 · 상태(신규/증가/감소/제외)가 함께 표시
- **8_ETF상세** : 7번 시트가 참조하는 원본(전 종목×ETF 보유 내역). 데이터가 커서 파일이 다소 큼
- **6_전체데이터** : 전 종목 집계 결과 (조회 시트가 참조하는 원본)
- (0_요약 : 기준일·방법론)

→ 주요 종목을 csv에 넣을 필요 없이 **'5_조회' 시트에서 바로 검색**하면 됩니다.
  (`주요종목.csv` 는 더 이상 사용 안 함. 지워도 됨.)

### 표의 숫자는 실제 엑셀 수식
1·2·3·4·6 시트의 **편입비중(%)·억 단위·비중변화**는 원본(원) 컬럼에서 `=ETF보유액/시가총액*100`
같은 **엑셀 수식으로 계산**됩니다(INDEX/MATCH·IF, 엑셀 2019/2020 호환). 셀을 클릭하면 계산식이 보입니다.

## 방법론
- 모든 ETF PDF에서 **실제 상장 개별주식인 구성종목만** 추립니다.
  (채권/선물/현금/타ETF 자동 제외, **혼합형 ETF의 개별 종목은 포함**)
- 종목별 ETF 보유액 = Σ ( ETF 순자산(NAV×상장좌수) × 그 ETF에서의 구성비중 )
- 편입비중(%) = 종목별 ETF 보유액 합 / 그 종목 시가총액 × 100

## 설정 (etf_holdings.py 상단 CONFIG)
- `BASE_DATE`    : 분석 기준일. `""` 이면 최근 영업일 자동.
- `COMPARE_DATE` : 비교 기준일. 기본 `"20251230"`.
- `WORKERS`      : 동시 요청 수(기본 8). 차단되면 낮추기.
- `TOP_N`        : 시트1·2 상위 개수 (`None` 이면 전체. 단 4_전체데이터는 항상 전 종목).

## 참고
- `krx_계정.txt` 에는 비밀번호가 들어가니 외부에 공유/업로드하지 마세요.
- '3_조회' 시트는 XLOOKUP이 아니라 INDEX/MATCH라 옛 엑셀에서도 동작합니다.
  종목코드는 6자리(예: 005930)로, 또는 종목명으로 입력하세요.

## 웹 뷰어 (index.html)
크롤러를 돌리면 엑셀과 함께 웹용 데이터도 출력됩니다:
- `data.json` : 모든 날짜의 종목 집계(가벼움, 날짜가 누적 병합됨)
- `detail_<날짜>.json` : 그 날짜의 ETF별 상세(무거움, 날짜별 분리 → 필요할 때만 로드)

`index.html` 을 브라우저로 열면(또는 호스팅하면) 종목 검색·시장 전체·ETF별 비교를 볼 수 있습니다.
- 로컬에서 더블클릭 시: 브라우저 보안(CORS)으로 json을 못 읽어 **내장 샘플**이 보입니다.
- **실데이터는 웹 호스팅(예: GitHub Pages) 상태에서** 표시됩니다 (3단계).
- 매일 자동 크롤링하면 data.json 에 날짜가 계속 쌓여 드롭다운에 추가됩니다.

## 실행 모드 (요약)
- **수동실행.bat** : 날짜를 여러 개 입력(띄어쓰기/콤마) → 그 날짜들 전부 크롤링.
  예) `20251230 20260430 20260529 20260625`. 엔터만 치면 최근 영업일 1개.
  + 엑셀 1개(최신 입력일 vs COMPARE_DATE) 생성.
- **매일실행.bat (자동, --auto)** : 당일(최근 영업일)만 크롤링 → 웹 데이터만 갱신(엑셀 X) → git push.
  작업 스케줄러로 평일 저녁 자동 실행. (스케줄러 동작 인수에 `auto` 필요)
- 비교(분석일 vs 비교시점)는 웹 페이지의 달력에서 고르는 방식이라, 크롤러는 "각 날짜 데이터"만 쌓으면 됩니다.
